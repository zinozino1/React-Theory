```js

1. 리액트에서 반복적인 내용을 효울적으로 보여주고 관리하는 법
-> 배열 고차함수를 필히 사용할 것.

** key
- 리액트에서 key는 컴포넌트 배열을 렌더링했을 때 어떤 원소에 변동이 있었는지 알아내려고 사용함. key 가 없다면 컴포넌트 배열 리스트를 순차적으로 비교하면서 변화를 감지하지만 key값이 있다면 이 값을 이용하여 어떤 변확가 일어났는지 더욱 빠르게 알아낼 수 있음
- key 값은 DOM 요소 뿐 아니라 컴포넌트 요소에 사용하는 것도 가능함.

1) arr.map((v, i, arr)=>{return v * v})

=> 콜백함수의 두번째 매개변수가 배열의 index임을 주의할 것, 꼭 return 값이 있음을 주의할 것
=> 데이터 배열을 컴포넌트 배열로 변환할 수 있음.

**** 주의
- '데이터'가 가진 고윳값을 key값으로 설정해야함. ex){id:1, title:'hi', desc:"ddd"} id 값.
- map함수의 index를 이용할 수도 있으나 이렇게 하면 배열이 변경될 때 효율적으로 리렌더링 되지 못함. 따라서 데이터 자체에 고유한 값이 없다면 어쩔 수 없이 map의 index를 활용해야하는 것

- id 값도 nextId로 state 형식으로 관리해주는 것이 좋다.

2) arr.concat(객체)

=> side effect가 없기 때문에 arr.push 대신 사용된다. state의 불변성을 유지.

3) arr.filter((v, i, arr)=>{return ...})

=> 배열의 특정 조건을 만족하는 원소들만 쉽게 분류할 수 있음.
=> map과 마찬가지로 return 있는 것 조심
=> none side effect




```
